Windows下Git+Hexo在GitHub上搭建博客站和项目站

发表于 2015-12-20   |   分类于 08工具

背景

这两天在做DC305的基础培训，在培训过程中发现许多人想知道软件中的英语都是什么意思。对此我想以后逐渐的翻译，但也不能等都翻译完了再公布出来，就想翻译一部分就发布一部分。问题就来了，怎么发布呢。用微信订阅号，总感觉不安全。我的想法是把微信公众号做为一个和学员联系的接口，培训的内容也好，课件也好都放在一个安全的地方。

针对上述问题我一开始想的用博客来存放培训内容和相关数据，但市面上的博客大多都有很多广告，最后无奈放弃。然后我发现了像简书一类的写作平台，找了好几个最后发现这些平台都有分享的性质，我其实并不想让学员或其他人把我的这些资料都拿走分享出去，所以也放弃我一方案。正当我感觉没有办法想就用微信公众平台的时候，突然间发现了可以用GitHub搭建博客！

发现了就去做。然后就是百度查资料。看了许多教程之类的，不是我想要的。后来发现了叶阳栩宁的Hexo系列教程。这就是我想要的，然后就按照教程说明来做。

教程

写这个文章也是怕以后自己需要在自己的电脑上安装，所以记录下来我这次的安装步骤，这里我就不会放图片了。

1. 搭建Hexo的Windows环境

安装Git

Git官网下载,默认安装

安装Node.js

NodeJs官网下载，默认安装

安装Hexo

在任何地方右键-Git Bash Here 打开Git的Bash，然后使用npm命令
npm install -g hexo

创建Hexo目录

目录创建在英文无空格目录下
然后在此目录下右键，打开Git Bash,或者在已打开的Bash里面，使用cd命令进入Hexo目录。

  • 本地网站初始化
    hexo init
  • 安装依赖环境
    npm install
  • 运行
    hexo server 或 hexo s
    然后打开本地网站看一下是否成功

2. 配置GitHub

GitHub是免费的，且没有流量和空间的限制，是个人博客和项目演示的地工具。

配置SSh key

配置SSh key是为了以后部署网站时不用重复输入GitHub账号的用户名和密码。

使用Git Bash，生成新的SSh key

然后连接按回车，ssh key会保存在默认位置

将生成的ssh key添加到GitHub上

执行如下命令
cd ~/.ssh/ && ls
可以看到有2个文件:id_rsa和id_rsa.pub
再执下面的命令
cat id_rsa.pub
把pub文件中的内容复制到下来
登录到GitHub网站，导航到: Account Settings —> SSH keys —> Add SSH keys
把key粘贴到里，名称随便写。

测试是否成功。需要过一小会，不会立刻生效。

ssh -T git@github.com
如果是以下提示：“Hi bright! You’ve successfully authenticated, but GitHub does not provide shell access.”
证明你配置成功了

3. 新建GitHub 仓库

这里需要说明的是，GitHub的Page有2种，且互不影响：

  • 增加库时，名称那里写 username.github.io。通过形如这样http://username.github.io的URL访问。
  • 增加库时，名称那里写 sitename。通过形如这样http://username.github.io/sitename的URL访问。

第1种是Hexo搭建Blog要用的，
第2种我用来放我的培训材料等数据。
创建完仓库即可，对于不是代码的内容，协议选择CCZ*即可。
